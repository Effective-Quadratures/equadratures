Base polynomial¶

The polynomial parent class.

class equadratures.poly.Poly(parameters, basis)[source]¶

The class defines a Poly object. It is the parent class to Polyreg, Polyint and Polycs; the only difference between its children are the way in which the coefficients are computed. This class is defined by a list of Parameter objects and a Basis.

Parameters:
  • parameters (Parameter) – A list of parameters.
  • basis (Basis) – A basis selected for the multivariate polynomial.

evaluatePolyGradFit(stackOfPoints)[source]¶

Evaluates the gradient of the polynomial approximation of a function (or model data) at prescribed points.

Parameters:
  • self (Poly) – An instance of the Poly class.
  • stackOfPoints (matrix) – A N-by-d matrix of points (can be unscaled) at which the polynomial gradient must be evaluated at.
Returns:

A d-by-N matrix of the gradients of the polynomial approximation.

Notes:

This function should not be confused with getPolynomialGradient(). The latter is only concerned with approximating what the multivariate polynomials gradient values are at prescribed points.

getQuadratureRule(options=None, number_of_points=None)[source]¶

Generates quadrature points and weights.

Parameters:
  • self (Poly) – An instance of the Poly class.
  • options (string) – Two options exist for this string. The user can use ‘qmc’ for a distribution specific Monte Carlo (QMC) or they can use ‘tensor grid’ for standard tensor product grid. Typically, if the number of dimensions is less than 8, the tensor grid is the default option selected.
  • number_of_points (int) – If QMC is chosen, specifies the number of quadrature points in each direction. Otherwise, this is ignored.
Returns:

A numpy array of quadrature points.
Returns:

A numpy array of quadrature weights.
